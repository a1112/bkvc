About

Version: 1.0.0
This readme will give an overview about the content of this repository. All integration samples requires that the used Visionary camera supports the GigE Vision protocol specifications. Currently supported devices:

  • Visionary-B Two

Prerequisites

Visual C++ Redistributable Package

To run these code examples which use GenICam binaries, you need to have the Visual C++ 2013 runtime environment installed, i.e., vc120. If you don’t have it installed you can run the executable in the VCRedist directory to install the English version. If you prefer another language, you can download the redistributable package from here.
Make sure to use the 64-bit version (x64).

Network connection to the device

  • Minimum 1000 Mbit/s Ethernet connection is required

  • The Ethernet interface which is used to connect to the camera should support jumbo frames

  • All Ethernet infrastructure between camera and your machine should also support jumbo frames

  • Without the use of jumbo frames, optimal network performance cannot be guaranteed

Python installation and usage

Most of the sample code (see more details below) is designed for use with Python 3.x (code is tested with Python 3.8, but might also work with newer versions). Python can be obtained from it’s offical website.

If a different version of Python is already installed, it is recommended to use Python’s virtualenv package, to create a separated environment with the desired Python version.
In the following example, the system has a Python 3.10 installation (which is already in use) and it shows how to setup a virtualenv with Python 3.8.

Sample scripts, tools and supporting libraries

Camera programming tutorials, sample scripts, helper tools as well as their supporting libraries are located in their respective language-specific subdirectories:

  • python - scripts accessing the cameras using Python language, by means of the Harvesters package.

  • cpp - preliminary minimal API to access the cameras from a C++ project, including examples.

  • common - common libraries shared across languages, in particular the GenICam GenApi implementation libraries and SICK’s GenTL Producer for GigE Vision cameras.

The individual samples and tools are described in dedicated sections below.

Python sample description

Target audience
All provided Python scripts (in the python/ subdirectory) are intended to be used by developers and integrators! It is assumed that basic Python is known, and one can understand (and adopt) the existing code.
Anything inside other folders, such as python/lib/ is not intended to be called directly. Those files contains classes and functions used by other scripts.

tutorial_harvesters.py : detailed commented walkthrough usage of Harvesters

It is recommended to view and understand this file first before starting your work in order to use GigE with Harvesters. This script is documented and shows step by step what is done.
The script also includes a few checks of the used environment, please read hints and warnings carefully if some occur!

ip_config.py : managing the IP configuration of the camera

The script allows to query and control the IP configuration of the camera - an important prerequisite for reliable use of the device. A camera running with IP address that does not match the subnet of the network card to which it is connected is not accessible. The main tasks supported by the script are

  • List devices currently visible to the system, including their identity information and current IP settings.

  • Force a temporary IP address (explicitly specified or automatically selected) to bring the device into the network card’s subnet. The temporary address does not survive power cycle.

  • Configure persistent settings specifying how the IP address will be configured during all future boots (specific IP selected, DHCP or LLA protocols to be used), optionally force restarting the IP config procedure.

Please read the script usage information (--help command line parameter) for further information.

sample_imu_data.py : detailed commented walkthrough usage of ImuBasic data

It is recommended to view and understand this file if you want to use the IMU data from the camera. The script shows how to enable and capture ImuBasic data and how to parse it. For demonstration, the data is printed in csv format.

Configuration files (TOML)

The recording script uses a parameter file for the configuration of the camera(s). This configuration file follows the TOML syntax, see: https://toml.io/en/
A sample configuration with detailed description can be found here: sample.cfg
It is recommended to read this file carefully (and copy and adopt it for your needs) before start using these scripts.

record.py : recording from GigEVision cameras

Can be used to record from single or multiple cameras. Recording will be done for a duration, number of frames or auto bracket mode.
Recording for multiple cameras will be done in a round-robin manner to avoid threading/bandwidth problems. This will lead to low overall framerate, but provides superior stability.

load_pickle.py : load data from recorded files

This sample script shows the principle how recorded data can be read from a pickle file, which was created by the use of the record.py script. It will loop over all contained frames, showing debug info and show the contained image in a plot. inspect the code to learn how the data is accessed.

load_pickle2mat.py : convert recording from pickle to matlab file

This sample script converts a provided pickle recording into a matlab file *.mat. This can be used for the analysis/processing of the image data in Matlab.

load_pickle2ply.py : convert recording from pickle to PLY point cloud

This sample script converts a provided pickle recording into a PLY (point cloud) file. It will also contain the color information (must be contained in the pickle recording). This can be used to inspect the point cloud of a singe capture. See below for tool recommendations.

load_pickle2ssr.py : convert recording from pickle to SICK Scan data Record

This sample script converts a provided pickle recording into a *.ssr file. This can be used to inspect the recording in SICK SopasET.

stream_cam2ros.py : stream connected Visionary camera over GigE to a ROS2 node

This sample script scans for connected Visionary cameras over the GigE interface and streams the camera image, depth image and pointcloud data via a ROS2 node. The streaming data can be further viewed and analyzed using a program such as Rviz2 or FoxGlove. Note that this requires a working ROS2 environment on the host PC. The necessary steps are then:

: activate the ROS 2 environment
c:\opt\ros\foxy\x64\setup.bat
:: activate the Gazebo simulation environment
c:\opt\ros\foxy\x64\share\gazebo\setup.bat
set "SDF_PATH=c:\opt\ros\foxy\x64\share\sdformat\1.6"

C++ sample description

Target audience
The GenIStream SDK and associated sample programs are provided in the cpp/ subdirectory and is intended to assist you with accessing the camera from a C++ based project.

The current version of the GenIStream SDK only offers a basic implementation for the Visionary-B Two product family. There is a recommended subset of the APIs that can be used with the device. Further details are provided in the cpp/GenIStreamSDK/README.txt. A fully generic GenIStream SDK is currently under development.

TutorialGenIStream.cpp : detailed commented walkthrough usage of GenIStream SDK

Extensive example demonstrating the entire workflow of SICK Visionary camera control using the Visionary GenIStream API. It is recommended to base Visionary camera integration in your C++ project on this example. The explanations covers also specific topics related to the Visionary camera itself and topic related to using GigE Vision protocol in general with satisfactory performance.

SampleIMUData.cpp : IMU data access in GenIStream SDK

Demonstrates how to enable, acquire and parse the IMU data in a GenIStream SDK based application.

Misc

View exported PLY files

As described above, pickle files can be exported into PLY format. To view PLY files we recommend:

  • Meshlab

  • CloudCompare

Storing buffers to disk in Pickle files

The samples use pickle to store and load camera data to disk. The maximum possible recording speed depends on your disk speed - however, modern SSD storage is fast enough to write all data, even if the camera runs with maximum framerate. Please be aware, that writing uncompressed image data requires big amounts of free disk space.
The Writer class uses whitelist objects, which defines what should be stored for each frame. The definition of the whitelist (see method _create_wl()) is also stored in the pickle file. Hence, the Reader can load a list of expected values and data directly from the individual pickle file. This mechanic makes it easy to add further parameter to be stored without breaking other recordings.

Troubleshooting

Create network traffic logs

Help us supporting you, by capturing the network traffic (while your issue appears) with Wireshark
This tool has a big community, you will find any kind of help/howto in the web.

Enable debug log in harvesters

For more detailed debugging, it is possible to use Python’s logging module to get debug logs from harvesters:

from harvesters.core import Harvester
import logging
logging.basicConfig(format='%(asctime)s %(message)s', level=logging.DEBUG)
with Harvester(logger=logging) as h:
  ... <your script as usual> ...
Enable GenTL Producer logs

In some cases (not necessary for normal operation) detailed debugging of the GenTL layer can be enabled. Please only enable this logging if you were asked to do so. It will be enabled by setting these environment variables:

SICK_GENTL_LOGGING=1
SICK_GENTL_LOGGING_LEVEL=99

On Windows systems, the tool DebugView can be used to fetch the enabled log messages.
On Linux systems, the enabled log messages will be printed to stderr

Multiple devices found

If some script exists with a message like multiple devices found please check if the device answer to a discovery command two times. One with read/write access and one without. If this is the case, we strongly recommend reviewing the IP address configuration of your setup. We recommend to not use multi homing: e.g. using two different IP addresses from different subnets on the same physical network interface.

Known Issues

  • Downloading any camera system logs is not supported yet

  • Enable only ImuBasic component and ChunkModeActive did not work with harvesters, because of a bug in harvesters implementation. Recommended to use GenIStreamSDK if needed.

  • Message- and Eventchannel do not work with harvesters, because of a bug in harvesters implementation. Recommended to use GenIStreamSDK if needed.
